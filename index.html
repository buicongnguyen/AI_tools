Efficiency system

Spend tokens on evidence, not repeated correction

1. Define “done”

Give the agent observable acceptance criteria, named files, boundaries, and exact verification commands.

2. Protect context

Keep instruction files concise, use focused tests, summarize noisy exploration, and start fresh when history stops helping.

3. Close the loop

Let the agent run tests, inspect screenshots, compare outputs, and fix discrepancies before asking you to review.

4. Systemize wins

Stable rules go in AGENTS.md or CLAUDE.md; repeatable workflows become skills; mandatory checks become hooks or CI.

Choose the smallest effective workflow

WorkEfficient pattern
Small obvious editImplement directly, run one focused check, inspect the diff.
Unfamiliar or multi-file featureExplore → plan → implement → verify.
Unclear bugReproduce → rank hypotheses → add regression test → smallest fix.
UI workReference → implement → browser/screenshot comparison → iterate.
Large repeated migrationTest 2–3 samples → refine prompt → bounded batch automation.
High-risk resultBuilder → fresh independent reviewer → human approval.
Software track

Idea to production

  1. Turn the user problem into acceptance tests.
  2. Trace architecture, interfaces, data, and trust boundaries.
  3. Build the smallest vertical slice.
  4. Run unit, integration, end-to-end, performance, and security checks.
  5. Debug from reproduction evidence, not guesses.
  6. Review, stage, deploy, observe, and preserve rollback.
Hardware track

Idea to fielded system

  1. Define performance, power, thermal, area, cost, and safety budgets.
  2. Document clocks, resets, protocols, register maps, and failure behavior.
  3. Model or simulate before committing to hardware.
  4. Build RTL, firmware, PCB, or prototype with self-checking tests.
  5. Bring up safely and correlate logs, traces, waveforms, and measurements.
  6. Stress, fault-test, pilot, update, monitor, and verify recovery.

Prompt contract

  1. Goal: observable outcome.
  2. Context: files, users, constraints.
  3. Scope: may/must not change.
  4. Acceptance criteria.
  5. Verification commands.
  6. Evidence-based output.

Debug loop

  1. Reproduce.
  2. Capture raw evidence.
  3. Rank hypotheses.
  4. Instrument one boundary.
  5. Add failing test.
  6. Smallest fix.
  7. Focused then broad tests.
  8. Review and monitor.

Idea gate

Proceed: ≥75 and no fatal risk.
Experiment: 55–74 or key assumptions untested.
Pause: <55 or unacceptable risk.

Definition of done

Criteria met, tests pass, diff reviewed, claims cited, risks visible, preview verified, rollback ready, and a human owns the final production decision.
