Prompt contract

  1. Goal: observable outcome.
  2. Context: files, users, constraints.
  3. Scope: may/must not change.
  4. Acceptance criteria.
  5. Verification commands.
  6. Evidence-based output.

Debug loop

  1. Reproduce.
  2. Capture raw evidence.
  3. Rank hypotheses.
  4. Instrument one boundary.
  5. Add failing test.
  6. Smallest fix.
  7. Focused then broad tests.
  8. Review and monitor.

Research ladder

  1. Official primary source.
  2. Independent reputable analysis.
  3. Community report as lead only.

Label facts, inference, estimates, conflicts, confidence, source dates, and verification date.

Test-set architecture

LayerWhat it provesEvidence
Happy pathPrimary user outcome worksAssertions + critical-flow smoke test
Boundary/invalidLimits fail safelyError/status assertions
Security/adversarialSecrets, permissions, and trust boundaries holdNegative tests + review
RegressionKnown failures stay fixedFrozen fixtures
Human rubricUsefulness, clarity, accessibilityBlind scored review

Deployment sequence

Human owner

Defines risk, approves preview and production.

→
AI agent

Implements, tests, reviews, and reports evidence.

→
CI + monitoring

Validates clean build, deploys version, checks health and rollback.

Idea gate

Proceed: ≥75 and no fatal risk.
Experiment: 55–74 or key assumptions untested.
Pause: <55 or unacceptable risk.

Definition of done

Criteria met, tests pass, diff reviewed, claims cited, risks visible, preview verified, rollback ready, and a human owns the final production decision.
